✓ BUILT & WORKING

Shipped features

All of the following are live, tested, and deployed.

Drawing

✎
Pen tool working
Pressure & tilt-aware ink with smooth curves. One input pipeline for mouse, touch, stylus & Apple Pencil, with palm rejection.
⌫
Eraser working
Erases strokes on contact with a circular hover ring showing exactly what a tap will remove.
◌
Lasso select & move working
Draw a lasso to select strokes, then drag to move them or press Delete/Backspace to remove the group.
●
Pen sizes & colours working
Three stroke widths, 8 preset colours, and a custom colour wheel.
▦
Paper styles working
Blank, Grid, Lined and Dot-grid backgrounds — cycle through with one button.

Typed text

T
Text tool working
Place, drag, and recolour editable text boxes anywhere on the canvas.
⌨
On-screen keyboard working
A tablet-friendly virtual keyboard for typing without hardware; Done to finish.
⎘
Physical typing & paste working
Type with a real keyboard or paste clipboard text straight onto the canvas.

Stylus AI

✦
On-device handwriting OCR working
Recognises handwriting in the browser (Tesseract WASM) — nothing is uploaded.
✦
Claude refinement working
Six one-tap actions via the Vercel AI Gateway: Polish, Fix grammar, Summarize, To-do list, Formal, Casual.
⧉
Copy / replace results working
Copy the refined text, regenerate, or drop it back onto the canvas.

Canvas navigation

⤢
Zoom & pan working
Pinch or ⌘-scroll to zoom toward the cursor, scroll to pan, plus a −/100%/+ & reset control cluster.

History & data

⇄
Undo / Redo / Clear working
Full history with keyboard shortcuts; Clear asks for confirmation before wiping ink & text.
🗀
Multi-document sidebar working
Create, switch, rename and delete unlimited notes from the sidebar.
⤓
Local-first storage working
Every note is saved in the browser — no account, works offline, privacy-friendly.

Export & hardware

🖼
Export PNG & PDF working
Save your work as an image or a document.
⌁
Bluetooth stylus working
Connect a Bluetooth stylus where the browser supports Web Bluetooth.
⎙
Scanmarker scanner working
Connect a Scanmarker pen-scanner; scanned text drops onto the canvas as a text box.

Performance & accessibility

⚡
Layered canvas rendering working
A static layer for committed ink + a cheap overlay for the live stroke — drawing no longer re-renders the whole document each frame.
♿
Accessibility fixes working
Proper <main> landmark, inert (non-focusable) closed sidebar, and pinch-zoom enabled.
✓
Test suite & CI build working
119 passing tests; clean typecheck, lint and production build.
⏳ PENDING / KNOWN GAPS

Not done yet

Open items — by design, by limitation, or simply not built.

⚡
OCR off the main thread to do
Drawing performance is fixed, but the Convert click still blocks ~3 s while the OCR engine runs synchronously. Moving Tesseract into a Web Worker would remove the freeze.
🔋
Scanmarker battery level limitation
WebHID doesn't expose the scanner's battery service, so the battery readout is a placeholder stub.
🎓
First-launch onboarding to do
Currently light-touch (placeholder hints + tooltips only). No guided 3-step tutorial for new users.
☁
Cloud sync / accounts not planned
Storage is local-only by design. No login, no cross-device sync.
⤒
Image / PDF import to do
Export works both ways out; there's no way to import an image or PDF onto the canvas.
🔒
Public access setting
Production is behind Vercel Deployment Protection (login required). Making it public is a project setting, not a code change.
