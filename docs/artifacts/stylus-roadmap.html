Phase 2 · Parity

Competitive parity

Days 31–60

Sync and PDF import are the two hardest items on the whole list — and both P0 blockers for named user segments. They go here, not later, because every Phase 3 feature is wasted without them.

02

Accounts + sync MVP

Implement auth and local-first sync — write locally, sync in background. Last-write-wins is acceptable for MVP; full conflict resolution isn't required yet.

High effort depends on 02a
03

PDF import + annotation

Integrate pdf.js. Render each PDF page as a background layer under a notebook page, using the existing per-page background template infrastructure.

High effort
05

Selection resize + rotate

Wire the existing decorative corner handles in SelectionToolbar.tsx to real scale/rotate transforms. Add hit-testing in geometry.ts. Commit as one undo step.

Medium effort

Phase 3 · Differentiation

The AI layer

Days 61–90

The stuff no competitor has. Handwriting search is the highest leverage item here — it makes the existing OCR investment pay off across the whole product, not just the Convert button.

04

Handwriting search

Run background OCR on save via the existing Tesseract.js pipeline. Store recognized text alongside strokes in the search index — no manual Convert step required.

Medium effort
13

AI auto-organization

On save, call /api/refine with a new prompt type to suggest tags and a folder for the current document. Surface as a dismissible suggestion.

Medium effort
14

AI page summaries

Medium effort
